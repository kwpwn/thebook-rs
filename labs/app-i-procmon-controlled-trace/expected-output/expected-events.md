# Expected Procmon Event Patterns

Exact ordering and extra metadata operations vary. Focus on operation classes and correlation fields.

## File system

Expected path prefix:

```text
%TEMP%\InternalsNoteBookProcmonLab
```

Likely operations:

- `CreateFile`
- `QueryBasicInformationFile`
- `WriteFile`
- `ReadFile`
- `CloseFile`
- `SetDispositionInformationFile` or delete-related operation

## Registry

Expected registry prefix:

```text
HKCU\Software\InternalsNoteBookProcmonLab
```

Likely operations:

- `RegCreateKey`
- `RegSetValue`
- `RegQueryValue`
- `RegCloseKey`
- `RegDeleteValue`
- `RegDeleteKey`

## Process attribution

Expected process:

```text
powershell.exe
```

or:

```text
pwsh.exe
```

Record the PID printed by the script and match it with Procmon's PID column.

## Useful fields

- Time of Day
- Process Name
- PID
- Operation
- Path
- Result
- Detail
- Stack, if enabled

## Expected interpretation

This is controlled activity. The correct conclusion is not "suspicious PowerShell"; it is:

> The trace observed expected file and registry operations generated by the controlled script. The observation validates Procmon filtering, artifact capture, and interpretation workflow.
